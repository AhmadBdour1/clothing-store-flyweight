How Flyweight Pattern Works Here

The Flyweight Pattern is a structural design pattern that optimizes memory usage by sharing common data (intrinsic state) across multiple objects. In this clothing store:

  • Intrinsic State (Shared): Type, Brand, Design, Color, Material, Price, Image URL - stored once in ClothingModel
  • Extrinsic State (Unique): Size, SKU, Stock Location, Quantity - stored in ClothingItem
  • Factory Pattern: ClothingModelFactory ensures the same model is reused instead of creating duplicates

When you add multiple items with the same model (e.g., Nike Blue T-Shirt in different sizes), only one ClothingModel object is created and shared across all items, significantly reducing memory consumption.
